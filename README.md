# .github — системный контур JD7

Этот каталог описывает и поддерживает единый тестовый и операционный контур для всей экосистемы **JD7 WORLD**.

## Цели

- Обеспечить единый стандарт задач для тестировщиков (Tester.001, Tester.002, ...).
- Централизовать сценарии тестирования и чек-листы.
- Подготовить основу для автоматизации (CI/CD, проверки, отчёты).

## Компоненты

- **ISSUE_TEMPLATE/**  
  Шаблоны задач для разных ролей тестирования:
  - `tester-001-bug-report.md` — базовый отчёт об ошибке.
  - `tester-002-feature-check.md` — проверка фич / модулей.
  - `tester-003-regression-suite.md` — регрессионные проверки.

- **workflows/**  
  (опционально) Автоматические проверки, запуск тестов, линтеры и т.п.

## Связь с TESTER-HUB

Все шаблоны и процессы в `.github` синхронизированы с модулем:

- `modules/TESTER-HUB/` — центр тестирования JD7 WORLD.
- Сценарии и чек-листы описываются в TESTER-HUB и используются при создании задач через шаблоны.

## Роли

- **Tester.001** — базовое функциональное тестирование, воспроизведение багов.
- **Tester.002** — расширенное тестирование фич, UX, сценариев.
- **Tester.N** — специализированные роли (нагрузка, безопасность, интеграции).

Каждый шаблон задачи привязан к конкретной роли и уровню детализации.
