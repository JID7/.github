Сценарии тестирования — Free JD7

1. Общий доступ и загрузка

Цель: Убедиться, что Free JD7 доступен, загружается корректно и не ломается на старте.

- Сценарий 1.1 — Загрузка главной страницы
  - Открыть Free JD7 в браузере.
  - Ожидаемо: страница загружается без ошибок, видны основные блоки (навигация, контент, футер).

- Сценарий 1.2 — Адаптивность
  - Проверить отображение на мобильном, планшете, десктопе.
  - Ожидаемо: элементы не «ломаются», текст читаем, навигация доступна.

2. Навигация по модулям JD7 WORLD

Цель: Проверить, что пользователь может свободно перемещаться по ключевым модулям.

- Сценарий 2.1 — Переход в SCHOOL
- Сценарий 2.2 — Переход в QUESTRUN
- Сценарий 2.3 — Переход в TESTER-HUB
- Сценарий 2.4 — Переход в JUSTICE / NFT / BOTS (если подключены)

Для каждого:
- Ожидаемо: переход без ошибок, корректные заголовки, понятная точка входа.

3. Пользовательский путь «новичка»

Цель: Смоделировать первый вход пользователя в Free JD7.

- Зайти как новый пользователь (без контекста).
- Пройти базовый путь: главная → SCHOOL → QUESTRUN (если доступно) → TESTER-HUB (как справка).
- Оценить:
  - Понятно ли, что это за мир?
  - Есть ли базовые подсказки / онбординг?
  - Нет ли «мёртвых» ссылок?

4. Стабильность и ошибки

- Проверить консоль браузера на наличие ошибок.
- Проверить поведение при:
  - медленном интернете,
  - перезагрузке страницы,
  - возврате назад (Back).

5. UX-заметки

Отдельный блок для субъективных наблюдений:
- Что вызывает путаницу?
- Где не хватает подсказок?
- Какие элементы хочется нажать, но они не кликабельны?
